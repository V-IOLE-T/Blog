# 首页改版与文章目录重构设计

- 日期：2026-03-30
- 状态：已确认，待进入实现计划
- 适用项目：Blog

## 1. 背景

当前站点首页直接展示分页文章列表，访问者进入站点后首先看到的是内容流，而不是站长本人。站点已经存在一个 `/about` 页面，但该页面与首页割裂，且与首页同时存在时会形成重复入口。

本次调整的目标是将首页重构为“个人主页式关于我”页面，将文章浏览入口迁移到新的 `/blog` 页面，并以“逻辑文件夹”的方式重组文章目录体验。

## 2. 目标

### 2.1 用户目标

1. 站点首页改为自我介绍页面，不再展示文章列表。
2. 原 `/about` 页面不再保留，避免重复入口。
3. 新增一个独立的文章目录页 `/blog`。
4. `/blog` 页面按逻辑文件夹组织文章，而不是按磁盘目录或旧分类字段组织。
5. 新增站内简历页 `/resume`。

### 2.2 产品目标

1. 让首页更像个人品牌入口页，而不是纯博客索引页。
2. 让文章体系只保留一套归属规则，避免 `categories` 与新目录体系并存。
3. 保持现有主题布局风格，优先在现有 Astro 结构上演进，而不是完全重做站点。

## 3. 非目标

以下内容不在本次范围内：

1. 不保留 `/about` 的兼容跳转或双入口结构。
2. 不保留旧的分类体系及其兼容层。
3. 不引入多层级文件夹、专题页、封面页等更重的内容系统。
4. 不将简历页实现为 PDF 下载页。

## 4. 最终信息架构

重构后的核心路由如下：

- `/`：个人主页式首页
- `/blog`：文章目录页
- `/resume`：站内简历页
- `/article/...`：文章详情页
- `/archives`：保留，作为时间维度浏览入口
- `/about`：移除
- `/categories/...`：移除

导航层级调整如下：

- `文章目录` 指向 `/blog`
- `动态` 指向 `/talking`
- `昔日` 指向 `/archives`
- `留言` 指向 `/message`
- 顶部左上角 `Home` 继续返回 `/`

## 5. 数据模型设计

### 5.1 文章 frontmatter

文章内容模型只保留一套归属规则：

- 删除 `categories`
- 新增 `folder`
- 新增 `summary`

推荐的 frontmatter 结构示例：

```yaml
title: Astro 首页重构实战
date: 2026-03-30 12:00:00
updated: 2026-03-30 12:30:00
id: astro-home-redesign
folder: frontend
summary: 将博客首页从文章列表改造成个人主页，并新增文章目录页。
tags:
  - Astro
  - 站点重构
cover: /assets/images/example.webp
recommend: true
top: false
hide: false
```

字段职责说明：

- `folder`：文章所属逻辑文件夹，是目录页唯一的归属依据。
- `summary`：目录页右侧文章列表展示的摘要文案。
- `tags`：保留，继续用于文章补充标记。

### 5.2 文件夹配置

新增统一的文件夹配置表，例如 `src/page_data/BlogFolders.ts`，用于集中定义：

- `slug`
- `name`
- `description`
- `order`

示例结构：

```ts
export default [
  {
    slug: "frontend",
    name: "前端工程",
    description: "记录前端框架、构建工具、性能优化与界面体验相关内容。",
    order: 1
  }
];
```

设计约束：

1. 文件夹页面展示不直接从文章中推导展示文案。
2. 排序由配置控制，而不是由文章数量或文件名控制。
3. 页面渲染只展示在配置表中已声明的文件夹。

## 6. 页面设计

### 6.1 首页 `/`

首页采用“个人主页式关于我”结构，保留站点现有头部、横幅、侧边栏和整体主题风格，不改造成完全独立的新视觉系统。

页面结构：

1. 首屏 Hero
2. 技术方向区
3. 项目 / 小站入口区
4. 补充介绍区

#### 6.1.1 首屏 Hero

首屏展示：

- 站长姓名或身份定位
- 一段精炼的自我介绍
- 两个主按钮：
  - `查看文章目录`
  - `CV / 个人简历`

首屏不放“联系我”按钮。

#### 6.1.2 技术方向区

以 3 到 5 个信息块展示核心关注方向，例如：

- 前端工程
- 边缘计算
- AI 应用
- 自托管
- DevOps

每个区块仅保留一句短说明，服务于快速浏览，不复制现有 about 页的大段内容。

#### 6.1.3 项目 / 小站入口区

用于承接站长已有项目与站点入口，帮助访问者快速理解“你做过什么”。

#### 6.1.4 补充介绍区

吸收原 `/about` 中更具个人风格的内容，但控制篇幅，使首页仍然以“介绍”和“导航”功能为主，而不是长文阅读页。

### 6.2 文章目录页 `/blog`

`/blog` 采用“目录树 + 右侧简介”的双栏结构。

桌面端布局：

- 左栏：文件夹列表
- 右栏：当前文件夹信息与文章列表

移动端布局：

- 顶部：文件夹切换区
- 下方：当前文件夹说明与文章列表

#### 6.2.1 左栏内容

左栏每个文件夹展示：

- 文件夹名称
- 文章数量
- 当前选中态

默认进入页面时，展示排序后的第一个文件夹。

#### 6.2.2 右栏内容

右栏展示当前文件夹的：

- 标题
- 简介
- 文章列表

每篇文章卡片展示：

- 标题
- 日期
- `summary`
- 可选标签 `tags`

点击后进入现有文章详情页。

#### 6.2.3 交互要求

1. 文件夹切换应为轻量交互，不引入复杂状态机。
2. 首屏必须有默认选中的文件夹，避免右侧空白。
3. 移动端不保留双栏，避免挤压文本。
4. 当前选中的文件夹写入 URL 查询参数，格式为 `/blog?folder=<slug>`。
5. 当 URL 中未提供 `folder` 参数时，默认展示排序后的第一个文件夹。
6. 当 URL 中 `folder` 参数不存在或非法时，回退到默认文件夹，不展示空态页面。

#### 6.2.4 列表排序与过滤

`/blog` 中的文章组织与计数遵循以下规则：

1. `hide: true` 的文章不参与目录页展示，也不计入文件夹文章数量。
2. 同一文件夹下，`top: true` 的文章排在前面。
3. 非置顶文章按 `date` 倒序排列。
4. `recommend` 仅作为其他模块的展示参考，不影响 `/blog` 列表排序。

### 6.3 简历页 `/resume`

`/resume` 为站内页面，不使用 PDF。

页面建议包含以下区块：

1. 个人简介
2. 技术技能
3. 代表项目 / 作品
4. 经历或长期关注方向
5. 外部链接

简历页风格应保持稳定、清晰、易读，比首页信息密度更高，但仍与站点整体主题保持一致。

## 7. 现有功能影响面

### 7.1 需要移除的内容

1. `/about` 页面入口与页面文件
2. `/categories/...` 页面及相关分类聚合逻辑
3. 侧边栏分类模块
4. 内容模型中的 `categories` 字段
5. 所有站内显式指向 `/about` 或分类页的入口

### 7.2 需要保留的内容

1. 文章详情页路由与渲染逻辑
2. 标签系统
3. 归档页 `/archives`
4. 推荐文章等与分类无关的侧边栏模块

## 8. 迁移策略

建议按以下顺序迁移：

1. 更新内容 schema，声明 `folder` 与 `summary`
2. 新增文件夹配置文件
3. 新增首页 `/`、目录页 `/blog`、简历页 `/resume`
4. 将首页路由从文章列表切换为新首页
5. 下线 `/about` 与 `/categories/...` 相关逻辑
6. 更新导航与侧边栏开关
7. 批量为现有文章补齐 `folder` 与 `summary`
8. 验证构建与关键页面展示

迁移原则：

- 不保留双轨兼容逻辑
- 不同时保留旧首页结构与新首页结构
- 不继续维持 `categories` 与 `folder` 共存

## 9. 异常与边界处理

### 9.1 内容边界

1. 若文章缺少 `folder`，构建阶段应直接暴露问题，而不是在运行时静默归入默认文件夹。
2. 若文章的 `folder` 未在文件夹配置表中声明，构建阶段应直接报错。
3. 若文章缺少 `summary`，允许在目录页中不展示摘要，仅展示标题、日期与其他可用信息。
4. 目录页不通过正文自动截取摘要。
5. 若某个配置文件夹下暂无文章，默认仍展示该文件夹，以保持目录结构稳定。

### 9.2 页面边界

1. `/blog` 首次加载不能出现“右侧无内容”的状态。
2. 移动端需要避免左栏目录过长导致首屏不可读。
3. 首页模块数量不应继续扩张，避免重新退化成长篇 about 页面。
4. `/about` 与 `/categories/...` 下线后，未再定义替代路由，按站点默认行为返回 404。

## 10. 测试与验证

实现阶段至少验证以下内容：

1. 首页 `/` 是否展示新的个人主页式内容。
2. `/blog` 是否按 `folder` 正确聚合文章。
3. `/resume` 是否可正常访问。
4. 导航中是否已移除 `关于`。
5. 站点中是否不再出现分类页入口。
6. 文章详情页是否仍可正常打开。
7. `astro build` 是否通过。
8. 桌面端与移动端是否不存在明显布局错位。
9. `/blog?folder=<slug>` 直达与刷新后是否仍能展示正确文件夹。
10. 非法 `folder` 查询参数是否按预期回退到默认文件夹。

如仓库已有自动化测试，应在实现计划中纳入；若没有，则至少完成构建验证和关键路径人工验证。

## 11. 实现建议摘要

推荐的实现方向如下：

1. 通过新页面承接新信息架构，而不是在原分页首页上做条件分支。
2. 将“逻辑文件夹”做成显式配置，而不是从文章内容动态推导。
3. 将首页、目录页、简历页都建立在现有 Layout 基础上，复用当前主题体系。
4. 删除旧分类逻辑，不保留兼容代码。
