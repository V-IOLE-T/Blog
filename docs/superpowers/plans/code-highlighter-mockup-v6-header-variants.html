Code Highlighter / Header Variants

保留 A 的安静感，只重做 header

这四版都基于 `Quiet Insert` 的整体比例和轻阴影，只改 header 的语法。重点是把它从“组件头”变成更像正文边注、编辑标记或文件题签。

Header 1 Margin Note
边注式

上层是极轻的 section label，下层才是文件名。语气最像正文边上的编辑注释，不像一个控件 title bar。

这版把 header 拆成两层，信息密度被拉开，读起来更像文章里的题签和边注。

Header 2 Title Slip
题签式

把文件名当正文里的小标题处理，语言和 copy 退到右下角。更像一张贴在代码页上的题签。

如果你希望文件名更像“这一段的标题”，这版的秩序更明确，但仍然很安静。

Header 3 Stamp Label
印章式

把 `snippet` 做成一个很轻的 stamp，文件名和 meta 放在同一水平层。更有记忆点，但仍然不重。

这版比前两版更有“Shiroi 自己的标记系统”味道，但风险是装饰语义会更明显。

Header 4 File First
文件优先

先看到文件名，再在下一行看到语言和操作。最接近“正文小节标题 + 元信息”的组织方式，也最自然。

如果你不喜欢 label 抢在最前面，这版会更顺。它几乎就是正文里的一个小标题，下面附带一行 meta。
