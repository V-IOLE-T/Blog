Focus Language

Quiet Input + Note Textarea

这次只比较 focus 的表达方式。基础表面、圆角、纸感、排版都不动，只替换“聚焦时怎么提气”这件事。 目标是保留安静的阅读感，不再用左侧高亮线去强调交互。

A Soft Border
最稳

只做边框加深和极淡染色，没有额外标记，最不打断正文。

focused
像纸面被轻轻点亮。 A1
Cmd+Enter
很安静，但可见性最弱。 124/500
气质最像正文里的自然输入，不像组件态控件。
风险如果页面整体层级太轻，focus 感知可能不够强。
B Paper Lift
纸页被托起

聚焦时不是加线，而是让纸页轻轻离开桌面一点，强调结构而不是强调颜色。

focused
更有“纸片被翻起”的感觉。 B1
Cmd+Enter
对 textarea 和 editor 友好。 124/500
气质最符合纸张隐喻，交互很自然。
风险如果 hover 也有浮起，就要小心和 focus 混淆。
C Topline Wash
编辑批注感

accent 不再出现在左边，而是退成顶部一根淡淡的批注线，和 codeblock 的 header 语言更接近。

focused
更像编辑标记，而不是表单高亮。 C1

和 codeblock 语系更近。 124/500

风险单行输入上比较优雅，长 textarea 里可能略微太轻。
D Inner Bottom Line
偏输入语义

把 focus 感知压到底部内部线条，比较像书写基线，不再抢四周边框。

focused
更偏传统输入逻辑，但做得更轻。 D1
Cmd+Enter
更像输入法语义，不像批注语义。 124/500
气质最接近常规输入组件，但比普通 underline 方案更内敛。
风险对多行编辑器有时会显得过于“表单”。
