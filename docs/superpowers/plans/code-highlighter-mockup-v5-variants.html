Code Highlighter / Variants

收掉阴影，往正文里再靠一步

这三版都基于上一稿，但明显减轻了阴影和卡片漂浮感。区别主要在于它们和正文的距离感：A 最贴正文，B 仍保留一点“附页”语义，C 则几乎像编辑插注。

Variant A Quiet Insert
最接近正文

保留完整 header，但整块几乎贴在文中，没有明显悬浮感。适合默认文章代码块。

正文继续往下读时，这一块像一张被轻轻压平的附页，不需要先意识到“这是一个组件”，再去读内容。

Variant B Pinned Slip
保留附页感

仍然像夹进文章里的独立纸片，但阴影已经明显收住，只留下轻微 warm tape 和更柔和的浮起。

如果你还想保留一点“这是一页摘录”的感觉，这版更合适。它和正文有层次，但不会像单独的工具窗那么割裂。

Variant C Editorial Frame
像编辑批注

几乎去掉“卡片”这层物理感，只保留上下边界和左侧批注气质。最像正文中的结构化摘录。

这版的重点不在卡片，而在版式。它更适合你希望代码块被视作段落级内容，而不是一张独立纸片的时候。
