Notes Decision Markers
what to keep
Typography 输入内容与 placeholder 都用 sans，避免 serif 带来的“伪文学化”过强气质。
Focus 优先用左侧细线或底线表达 focus，配 4% 到 6% 的 accent 染色即可。
Error 错误像编辑批注，不做整块红底，也不做夸张 warning 状态。
Textarea 建议默认比 Input 更有纸感，这样长文本输入在页面里有合理的语义重量。
API 后续实现时把 label / hint / error / counter 提升到 `FieldShell`，避免 `Input` 与 `Textarea` 各自维护状态 UI。
