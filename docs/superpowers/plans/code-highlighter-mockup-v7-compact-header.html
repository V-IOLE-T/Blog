这一版直接沿你选的 `Header 3` 收：去掉 `snippet` 装饰，header 改成紧凑 sans，`language` 用 Material icon 挂到文件名左侧，`copy` 只保留 icon。下面同时放出无 filename 的两种 UI case。

With Filename Compact File Header
主方案

header 只保留一个紧凑信息层：`language icon + filename` 在左，`copy icon` 在右，语言文本退成弱提示，不再和 copy 挤在一起。

文件名存在时，最重要的是让它像代码页标题，而不是独立的组件头。这里的 header 已经压到接近正文注脚的尺度。

No Filename A Inline Meta Row
默认无文件名

没有 filename 时，不再硬保留“标题位”。直接退化成一条更短的 meta row：左边语言，右边 copy，整体还在正常文流里。

这版最稳。它保留 header 节奏，但明显比有文件名场景更轻，不会出现“空标题”的感觉。

No Filename B Headerless Utility
更内敛

彻底拿掉 header，语言和 copy 变成 code surface 上方的轻量工具标记。更像正文内嵌代码摘录，而不是一个“块级组件”。

如果你希望无文件名场景几乎并入正文，这版会更自然。但它的控制感更弱，适合内容优先的地方。
